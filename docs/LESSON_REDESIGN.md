> Дата планирования: 2026-06-14. Аудитория: дети 12–16 лет, **никогда не
> программировали**. Платформа - локальная, без сервера/сборки/библиотек
> (только HTML/CSS/JS), открывается в браузере.
>
> Этот файл - **источник истины по новому контенту и связанным доработкам
> платформы**. Статусы: `[ ]` запланировано, `[~]` в работе, `[x]` сделано.

## 0. Зачем переделываем (запрос пользователя)

1. **Постоянная наглядность.** Сейчас ребёнок жмёт «Запустить» ~1 раз за шаг.
   Надо: **много мелких итераций** «маленькая правка → ▶ Запустить → смотрю
   результат» - после каждого изменения видно, что поменялось.
2. **Время.** Сейчас урок проходится за ~30 мин. Цель - **~2 часа** реальной
   практики (ребёнок «примеряет профессию разработчика»).
3. **Прогрессия навыков.** Сначала ребёнок **пишет теги и текст руками** (без
   Emmet, без кнопок), потом - кнопки «Вставить» + заполнение меток, потом - сам
   пишет CSS-свойства/селекторы, и только **через несколько шагов** вводим Emmet.
4. **Связь со 2-м уроком (кибербез, тоже ~2ч).** В сайт урока 1 **намеренно**
   закладываем уязвимости; на уроке 2 ребёнок ищет/эксплуатирует их на **своём же
   сайте**: XSS в комментариях, небезопасный вход, секреты в localStorage/коде,
   подмена через URL/#hash (+ как можно больше практики).

## 1. Найденный баг: неверный `<title>` во вкладке «Открыть в новой вкладке»

- **Симптом:** при «🔗 Открыть в новой вкладке» в названии вкладки всегда
  «Мой интернет-магазин - просмотр», а не `<title>` из кода ученика.
- **Причина:** сайт ученика рендерится **внутри `<iframe>`** на `preview.html`.
  Название вкладки браузера берётся из `<title>` **самой `preview.html`** (там он
  зашит), а `<title>` внутри iframe на вкладку **не влияет** (стандарт браузера).
- [x] **Исправление (P1):** в `preview.html` в `render()` вытаскивать `<title>`
  из документа ученика (regex `/<title>([\s\S]*?)<\/title>/i` по строке `doc`) и
  присваивать `document.title` (если title нет - дефолт). Тогда сценарий шага 0
  работает: пустая страница, но в названии вкладки - title ученика; поменял
  `<title>` → ▶ Запустить → название вкладки изменилось.

## 4. Уязвимые блоки урока 1 (строим намеренно, ломаем на уроке 2)

Ребёнок строит их как обычные фичи, **не зная**, что они дырявые. Код нарочно
небезопасный. Важно: iframe превью имеет `allow-same-origin` + `allow-scripts` и
`srcdoc` → его origin = origin платформы, значит **`localStorage` в сайте ученика
= тот же `localStorage`**, а `alert()` работает (`allow-modals`). Это и делает
демонстрации XSS/секретов наглядными.

1. **Вход (H1) - небезопасная аутентификация + секрет в коде.**
   - HTML: форма (логин, пароль, «Войти») + блок «Привет, ИМЯ» (скрыт).
   - JS: проверка **захардкоженных** логина/пароля прямо в main.js
     (напр. `admin` / `megapass123`); при успехе - `localStorage.setItem('isLoggedIn','true')`,
     `localStorage.setItem('currentUser', логин)`, приветствие через **`innerHTML`**
     (`'Привет, ' + user`). HTML-комментарий-«подсказка» с «секретом» рядом.
   - Уроку 2 даёт: найти пароль в коде; обойти вход через консоль
     (`localStorage.setItem('isLoggedIn','true')`); XSS через имя.

2. **Комментарии (I1) - Stored XSS через `innerHTML`.**
   - HTML: поле ввода + «Отправить» + `<div id="comments">`.
   - JS: по клику `comments.innerHTML += '<div class="cmt">' + text + '</div>'`
     (БЕЗ экранирования) и сохранение списка в `localStorage`.
   - Уроку 2 даёт: payload `<img src=x onerror="alert('XSS')">` исполняется;
     понять, почему `<script>` через innerHTML НЕ исполняется, а `<img onerror>` -
     да; «кража» данных: `onerror="alert(localStorage.getItem('currentUser'))"`.

3. **Секрет/повышение прав - `is_admin` в localStorage + «админ-панель».**
   - JS: при загрузке читает `localStorage.is_admin` (по умолчанию `'false'`);
     если `'true'` - показывает скрытую «🛠 Админ-панель» с «секретным» текстом/флагом.
   - Уроку 2 даёт: в DevTools → Application поменять `is_admin` на `true` →
     перезагрузка → панель появилась (нарушение контроля доступа на клиенте).

4. **Приветствие из URL `#hash` - Reflected XSS.**
   - JS: `hello.innerHTML = 'Привет, ' + decodeURIComponent(location.hash.slice(1))`.
   - Уроку 2 даёт: ссылка `…#<img src=x onerror=alert(1)>` → отражённый XSS;
     обсудить, как такую ссылку «присылают жертве».

> Все 4 фичи строятся в уроке 1 (шаги H1, I1 + части секрета/hash встроить в H1/G).
> Точные сниппеты допишем при реализации (раздел «черновики сниппетов» ниже).

### Дополнительные идеи (по принципу «как можно больше», тоже с починкой)
- **Цена в DOM (Elements):** изменить цену товара в Elements → показать, что
  клиентская цена «не настоящая». Фикс-обсуждение: сервер обязан перепроверять
  цену при оплате.
- **`disabled`-поле/кнопка:** снять `disabled` в Elements и нажать. Фикс:
  проверка прав/состояния не должна полагаться только на `disabled`.
- **«Промокод» в JS:** найти секретный промокод в коде. Фикс: убрать секрет с
  клиента.
- **localStorage-тамперинг:** подделать счётчик корзины/баланс. Фикс: не хранить
  «доверенные» значения на клиенте.
- Каждую доп-задачу тоже вести в формате «найди → объясни своими словами →
  почини → проверь».

## 6. Открытые решения (требуют подтверждения при реализации)

## 7. Критерии приёмки

**Урок 1:**
- [ ] Прохождение занимает ~1.5–2 ч у новичка; «Запустить» используется **много
      раз** (несколько раз на шаг), результат виден после каждой правки.
- [ ] Видна прогрессия: руками → кнопки+метки → свой CSS → Emmet → JS.
- [ ] Title во вкладке = title ученика (P1).
- [ ] Кнопка CSS появляется только после вставки HTML того же шага (P2).
- [ ] Ручные шаги завершаются кнопкой «✓ Я сделал» и двигают прогресс (P3).
- [ ] Каркас - через Emmet «!» на шаге 0; общий Emmet (аббревиатуры `>`,`+`,`*`)
      вводится на шаге «Товары» (P4 - без доработок платформы).
- [ ] Построены 4 уязвимые фичи (вход, комментарии, секрет/админ, hash).
- [ ] Не сломаны: подсветка, история, чипы, консоль, ZIP, раскладка, адаптив.

**Урок 2:**
- [ ] ~2 ч практики; каждая из 4 уязвимостей реально находится/эксплуатируется
      ребёнком на своём сайте, без бэкенда.
- [ ] **КАЖДАЯ уязвимость проходится по циклу найди → почини → проверь:**
      ребёнок правит код, ▶ Запустить, эксплойт больше **не** срабатывает.
- [ ] Этический фрейминг («ломаем только своё»).
