# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это за проект

Полностью локальная (без сервера, без сборки, без внешних библиотек) веб-платформа
для обучения детей программированию. Один предустановленный многошаговый урок
«Собери свой интернет-магазин» (кнопки «Вставить блок» → «Запустить» → «измени сам»).

Структура файлов:
- `index.html` — разметка платформы; подключает все стили и скрипты.
- `css/` — стили, разнесённые по модулям (`base`, `layout`, `buttons`, `editor`,
  `markdown`, `preview`, `components`, `responsive`). Подключаются `<link>`-ами
  по порядку каскада.
- `js/` — логика, разнесённая по модулям (**классические** скрипты, НЕ ES-модули —
  чтобы работало и из `file://`; подключаются `<script defer>` по порядку,
  `main.js` — точка входа последней). Свой мини-Emmet — `js/emmet.js`.
- `preview.html` — отдельная вкладка-зеркало просмотра.

Всё открывается прямо в браузере, без сборки. Подробнее о модулях — в
`docs/ARCHITECTURE.md` → «Структура исходников (модули js/ и css/)».

Язык UI и общения с пользователем — **русский**.

## ВАЖНО: читать перед началом работы

Чтобы не повторять уже принятые решения и не наступать на известные грабли,
**в начале каждой сессии прочитай** (в этом порядке):

1. `docs/PLAN.md` — план разработки и текущий статус задач (что сделано / в работе / осталось).
2. `docs/ARCHITECTURE.md` — архитектура, ключевые решения и обоснования.
3. `docs/PITFALLS.md` — частые ошибки и готовые решения.

**В конце каждой сессии обнови** эти файлы: отметь выполненные задачи в `PLAN.md`,
допиши новые решения в `ARCHITECTURE.md`, занеси встреченные проблемы в `PITFALLS.md`.

## Запуск

Сборки и тестов нет. Открыть `index.html` в браузере, либо отдать каталог статикой
(`python3 -m http.server`) — второе предпочтительнее, т.к. `srcdoc` + `sandbox`
ведут себя стабильнее по HTTP, чем из `file://`.

## Полная спецификация
