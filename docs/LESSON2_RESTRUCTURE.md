## 0. ГЛАВНОЕ ограничение реализации

Урок 2 (`js/lesson-data-security.js`) **СГЕНЕРИРОВАН**, его НЕ правят руками. Шаги
заданы как список Python-словарей в **`scripts/build_lesson2.py`** (переменная
`steps`, через `steps.append({...})`). Поэтому:

1. Все правки шагов делаем в **`scripts/build_lesson2.py`**.
2. Пересборка: `python3 scripts/build_vuln_site.py` → `python3 scripts/build_lesson2.py`.
3. Вшитый сайт `VULN_HTML/CSS/JS` должен остаться **байт-в-байт прежним** (мы трогаем
   только массив `steps`, всё что после `const lesson = {`). Проверить diff по `VULN_*`.
4. После генерации: `node --check js/lesson-data-security.js`.

Поля одного шага (см. `step_js` в скрипте): `title`, `time`, `goalMd`, `actionMd`,
`taskMd`, `hintMd`, `doneMd`. Все шаги — `manual:true` (кнопка «✓ Я выполнил этот
шаг»). Прогресс «N из M» считается по числу шагов автоматически — увеличение числа
шагов ничего не ломает в движке (`lesson-render.js` не трогаем).

## 2. Принцип дробления

**Один шаг = один смысловой блок ≤ ~10–12 минут.** Базовая единица урока 2 — цикл
🔴 Найди/сломай → 🟢 Почини → ✅ Проверь. Дробим так:

- **🔴 «Найди/сломай» — отдельный шаг.** Ребёнок сперва своими глазами видит
  опасность (атака срабатывает), нажимает «✓ Я выполнил» и идёт чинить.
- **🟢 «Почини» + ✅ «Проверь» — следующий шаг** (починка маленькая без проверки
  бессмысленна, поэтому держим их вместе: поправил код → ▶ Запустить → повтори
  атаку → убедись, что не работает).
- Если внутри одной «дыры» несколько РАЗНЫХ мест/атак (как в Дыре №1) — каждое
  место становится своей парой шагов 🔴/🟢.

Итог ≈ **18–20 шагов** вместо 9 (см. таблицу в §4). Прогресс «N из 20».

## 3. Формат «пунктов действия» (в КАЖДОМ шаге)

В начало `taskMd` каждого шага добавляем стандартный блок **«📋 Пункты действия»** —
короткий нумерованный чек-лист (3–6 пунктов) из коротких повелительных фраз. Это
«карта шага»: ребёнок видит весь маршрут целиком, прежде чем читать подробности.

Под чек-листом — прежний подробный разбор «клик за кликом» (он остаётся, просто
короче на шаг, потому что шаги мельче). Структура `taskMd`:

```
📋 **Пункты действия** (что сделать в этом шаге):
1. Зарегистрируйся на сайте.
2. Войди под своим логином.
3. Открой комментарии у любого товара.
4. Вставь в комментарий «ловушку» и отправь.
5. Убедись, что выскочило окошко — это XSS.

---

🔧 **Подробно, по шагам:**
… (существующий текст «клик за кликом») …
```

**Реализация формата — без правок движка/CSS:** блок «📋 Пункты действия» — это
обычный markdown (жирный заголовок + нумерованный список), который уже умеет
рендерить `js/markdown.js`. Новые поля шага НЕ вводим (не трогаем `step_js`,
`lesson-render.js`, CSS) — всё живёт внутри `taskMd`.

> Альтернатива (НЕ берём пока): отдельное поле `actionsMd` + свой стиль
> `.action-items` в `css/markdown.css`. Красивее визуально, но требует правок ядра
> (общего для всех уроков) и `step_js`. Если по итогам ручной проверки захочется
> выделить чек-лист рамкой/фоном — вернёмся к этому отдельной задачей.
