## 1. Как урок встраивается в текущую архитектуру

**Решение по модели данных: расширяем объект `lesson` полем `steps[]`** (а НЕ
вводим отдельный механизм «много уроков» и не делаем селектор уроков).

Обоснование:
- Платформа намеренно однопользовательская: «один предустановленный урок»
  (`ARCHITECTURE.md`, «Чего НЕ делаем»). Селектор/роутинг уроков — вне scope.
- Многошаговость — это свойство одного урока, а не множества уроков. Значит,
  естественно описать её внутри существующего `lesson`, сохранив принцип
  «источник истины — объект `lesson` в `main.js`».
- Минимум изменений: переиспользуем `markdownToHtml`, подсветку, autosave,

Новый урок **заменяет** старый «Измени цвет текста» (активный урок всегда один).

## 2. Минимальные доработки платформы

Только необходимое для механизма «Вставить блок N» и шагов:
1. Рендер шагов в панели задания (карточки с кнопкой вставки).
2. Вставка сниппета в нужную вкладку (append в `value` + `updateHighlight` + autosave).
3. Индикатор прогресса «N из M» и пометка «готово» у выполненного шага.
4. **Линейная разблокировка**: активна кнопка только следующего невыполненного
   шага (гарантирует правильный порядок разметки и ведёт ребёнка по шагам).
5. Откат: общая кнопка «Начать заново» очищает код и прогресс шагов.

Без лишнего: нет отдельных «откатов по одному шагу», нет автозапуска (по ТЗ
ребёнок сам жмёт «Запустить»), нет проверки правильности правок.

## 5. Риски и грабли (сверено с PITFALLS.md)

- Внешние картинки нарушают «только локально» → используем **эмодзи**, не URL.
- Вставка через `value` могла бы порвать подсветку/автосейв → после вставки явно
  зовём `updateHighlight` + autosave (как и при `input`).
- `</script>` в JS-сниппете рвёт документ iframe → уже экранируется в `updateIframe`
  (PITFALLS); наши сниппеты его не содержат, но защита на месте.
- Вставка не по порядку дала бы неверный визуальный порядок → линейная
  разблокировка кнопок.
- Markdown-поля с инлайн-`код` внутри JS-строк → короткие поля пишем в двойных
  кавычках (бэктики литеральны), многострочные сниппеты — в шаблонных литералах
  (без бэктиков внутри).
- Повторная вставка одного блока (дубли) → кнопка после вставки становится
  «✓ добавлено» и блокируется.

- После каждого шага сайт визуально «вырастает» (шапка → баннер → товары →
  подвал → рабочая корзина).
- Прогресс и код сохраняются между перезагрузками; «Начать заново» сбрасывает.
- Не сломаны: подсветка, Tab/Emmet, консоль, скачивание ZIP, тосты, адаптив.
