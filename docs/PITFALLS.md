# Частые ошибки и решения

Журнал граблей, чтобы не наступать на них повторно в новых сессиях.
Формат записи: **Проблема → Причина → Решение**. Добавляй новые сверху раздела.

## Markdown-парсер

### `<` `>` `&` из описания ломают разметку
- **Причина:** текст вставляется через `innerHTML`; необработанные символы трактуются как теги/сущности.
- **Решение:** экранировать `& < >` (`&amp; &lt; &gt;`) **в самом начале**, до применения любых правил Markdown.

### `*курсив*` конфликтует со списками `* пункт` и с `**жирным**`
- **Причина:** один символ `*` используется и для списков, и для курсива; жадный регэксп курсива «съедает» звёздочки жирного.
- **Решение:** обрабатывать построчно. Сначала строки-заголовки (`#`), затем строки-списки
  (строка начинается с `- ` или `* `). Инлайн-замены в порядке: `` `код` `` → `**жирный**`
  → `*курсив*`. Для жирного и курсива использовать нежадные регэкспы (`\*\*(.+?)\*\*`, затем `\*(.+?)\*`).

### Списки рендерятся как отдельные `<ul>` на каждый пункт
- **Причина:** наивная построчная замена оборачивает каждый `- item` в свой `<ul>`.
- **Решение:** группировать подряд идущие строки-пункты в один `<ul>`; закрывать `<ul>`,
  когда встретилась не-списковая строка.

## iframe / превью

### `document.write` даёт пустой/мигающий iframe или гонку
- **Причина:** доступ к `iframe.contentDocument` при пустом `src` ненадёжен; `open/write/close` конфликтует с sandbox.
- **Решение:** использовать `iframe.srcdoc = fullHtml`. Декларативно, без обращения к contentDocument.

### Скрипт пользователя не выполняется в iframe
- **Причина:** в `sandbox` нет `allow-scripts`.
- **Решение:** атрибут iframe — `sandbox="allow-same-origin allow-scripts allow-popups allow-forms"` (как в ТЗ).

### `</script>` в пользовательском JS рвёт документ iframe
- **Причина:** строка `</script>` внутри инлайнового `<script>` закрывает тег раньше времени.
- **Решение:** при сборке документа экранировать `</script>` → `<\/script>` в коде JS
  (и при желании в HTML), либо помнить про это ограничение. (Низкий приоритет для урока про цвет, но фиксируем.)

## localStorage

### Исключение при доступе к localStorage (приватный режим / отключённый storage)
- **Причина:** в некоторых режимах обращение к `localStorage` кидает `SecurityError`.
- **Решение:** оборачивать чтение/запись в `try/catch`; при ошибке вести себя как «данных нет».

### После «Сброса» старый код возвращается после перезагрузки
- **Причина:** сбросили textarea, но не удалили ключ из localStorage.
- **Решение:** в `resetToTemplate()` обязательно `localStorage.removeItem('savedLessonCode')`
  (или перезапись initial-значениями) перед/после обновления редакторов.

## Скачивание файлов

### Память течёт / ссылка не скачивается
- **Причина:** не освобождён objectURL или ссылка не добавлена в DOM в некоторых браузерах.
- **Решение:** `URL.createObjectURL(blob)` → создать `<a>` с `download`, добавить в DOM,
  `click()`, удалить из DOM, затем `URL.revokeObjectURL(url)`.

## Прочее

### `defer` vs порядок инициализации
- **Причина:** если `main.js` без `defer` и подключён в `<head>`, DOM ещё не готов.
- **Решение:** подключать с `defer` или перед `</body>`; всю инициализацию — в обработчике `DOMContentLoaded`.
