## 1. Как урок встраивается в текущую архитектуру

**Решение по модели данных: расширяем объект `lesson` полем `steps[]`** (а НЕ
вводим отдельный механизм «много уроков» и не делаем селектор уроков).

- Платформа намеренно однопользовательская: «один предустановленный урок»

- Многошаговость — это свойство одного урока, а не множества уроков. Значит,
  естественно описать её внутри существующего `lesson`, сохранив принцип

Новый урок **заменяет** старый «Измени цвет текста» (активный урок всегда один).

## 2. Минимальные доработки платформы

Только необходимое для механизма «Вставить блок N» и шагов:
1. Рендер шагов в панели задания (карточки с кнопкой вставки).
2. Вставка сниппета в нужную вкладку (append в `value` + `updateHighlight` + autosave).
3. Индикатор прогресса «N из M» и пометка «готово» у выполненного шага.
4. **Линейная разблокировка**: активна кнопка только следующего невыполненного
   шага (гарантирует правильный порядок разметки и ведёт ребёнка по шагам).
5. Откат: общая кнопка «Начать заново» очищает код и прогресс шагов.

Без лишнего: нет отдельных «откатов по одному шагу», нет автозапуска (по ТЗ
ребёнок сам жмёт «Запустить»), нет проверки правильности правок.

## 5. Риски и грабли (сверено с PITFALLS.md)

- Внешние картинки нарушают «только локально» → используем **эмодзи**, не URL.
- Вставка через `value` могла бы порвать подсветку/автосейв → после вставки явно
  зовём `updateHighlight` + autosave (как и при `input`).
- `</script>` в JS-сниппете рвёт документ iframe → уже экранируется в `updateIframe`
  (PITFALLS); наши сниппеты его не содержат, но защита на месте.
- Вставка не по порядку дала бы неверный визуальный порядок → линейная
  разблокировка кнопок.
- Markdown-поля с инлайн-`код` внутри JS-строк → короткие поля пишем в двойных
  кавычках (бэктики литеральны), многострочные сниппеты — в шаблонных литералах
  (без бэктиков внутри).
- Повторная вставка одного блока (дубли) → кнопка после вставки становится
  «✓ добавлено» и блокируется.

  подвал → рабочая корзина).
- Прогресс и код сохраняются между перезагрузками; «Начать заново» сбрасывает.
- Не сломаны: подсветка, Tab/Emmet, консоль, скачивание ZIP, тосты, адаптив.

# Урок 3 «Аналитик данных и создатель ИИ» — KNN-симулятор (2026-06-20)

Третья «профессия» дуги: разработчик (урок 1) → этичный хакер (урок 2) →
**аналитик данных / создатель ИИ** (урок 3). В отличие от уроков 1–2 (редакторные
шаги внутри платформы prog_sim) — это **отдельное самостоятельное приложение** в
подпапке `lesson3/`: «просто открыть index.html». Без сервера, без сборки, без
библиотек; классические `<script>` (не ES-модули).

Суть занятия (~1 час): ребёнок ставит точки двух классов (🔵/🔴), крутит ползунок
`k`, жмёт «Обучить» → видит тепловую карту KNN, проходит секретный тест на 5
точках (Accuracy), экспортирует/импортирует датасет.

Структура `lesson3/`:
- `index.html` — разметка + гид из 4 шагов, подключение стилей/скриптов.
- `style.css` — тёмная тема (#12151e), неон (#00d4ff / #ff4757), свечение точек.
- `js/config.js` — все константы (цвета, размеры, k, прозрачность heatmap).
- `js/knn.js` — `euclidean`, `knnPredict` (голосование, ничья → ближайший,
  уверенность = доля голосов 0.5..1).
- `js/tests.js` — `generateTestPoints` / `classifyTestPoints` / `countCorrect`.
- `js/rendering.js` — сетка, тепловая карта (20×20, alpha ∝ уверенность), точки
  со свечением (shadowBlur), нумерованные тестовые точки.
- `js/app.js` — контроллер (состояние, события, клики ЛКМ/ПКМ, экспорт/импорт).

Реализованные требования: canvas 600×600; ЛКМ/ПКМ + переключатель цвета; ползунок
k 1–15; «Обучить» (heatmap пересчитывается; при `trained` k обновляет карту
вживую); «Стартовый датасет» (30 точек, диагональ); «Секретный тест» (5 серых +
выпадашки + «Показать ответы» с подсветкой верных/неверных и Accuracy); «Очистить
всё»; «Экспорт»/«Импорт» JSON; статистика (точек 🔵/🔴, точность теста).

Решения:
- **«Обучить» осмыслен:** добавление/импорт точек ставит `trained=false` (карта
  гаснет) — ребёнок видит, что после смены данных модель надо переобучить.
  А вот ползунок `k` при уже обученной модели обновляет карту мгновенно (видно
  влияние k без лишних кликов).
- Heatmap — снимок по сетке центров ячеек; точки рисуются поверх живьём.
- Тестовые точки серые до проверки, после — цвет предсказания KNN + подпись №.

- [x] Проверка логики в Node (vm): KNN (3-4-5, conf=1, k>n, ничья, пусто),
  tests (генерация в границах, classify, countCorrect — 5/3/0), heatmap-alpha в
