# Частые ошибки и решения

Журнал граблей, чтобы не наступать на них повторно в новых сессиях.
Формат записи: **Проблема → Причина → Решение**. Добавляй новые сверху раздела.

## Markdown-парсер

### `<` `>` `&` из описания ломают разметку
- **Причина:** текст вставляется через `innerHTML`; необработанные символы трактуются как теги/сущности.
- **Решение:** экранировать `& < >` (`&amp; &lt; &gt;`) **в самом начале**, до применения любых правил Markdown.

### `*курсив*` конфликтует со списками `* пункт` и с `**жирным**`
- **Причина:** один символ `*` используется и для списков, и для курсива; жадный регэксп курсива «съедает» звёздочки жирного.
- **Решение:** обрабатывать построчно. Сначала строки-заголовки (`#`), затем строки-списки
  (строка начинается с `- ` или `* `). Инлайн-замены в порядке: `` `код` `` → `**жирный**`
  → `*курсив*`. Для жирного и курсива использовать нежадные регэкспы (`\*\*(.+?)\*\*`, затем `\*(.+?)\*`).

### Списки рендерятся как отдельные `<ul>` на каждый пункт
- **Причина:** наивная построчная замена оборачивает каждый `- item` в свой `<ul>`.
- **Решение:** группировать подряд идущие строки-пункты в один `<ul>`; закрывать `<ul>`,
  когда встретилась не-списковая строка.

## iframe / превью

### `document.write` даёт пустой/мигающий iframe или гонку
- **Причина:** доступ к `iframe.contentDocument` при пустом `src` ненадёжен; `open/write/close` конфликтует с sandbox.
- **Решение:** использовать `iframe.srcdoc = fullHtml`. Декларативно, без обращения к contentDocument.

### Скрипт пользователя не выполняется в iframe
- **Причина:** в `sandbox` нет `allow-scripts`.
- **Решение:** атрибут iframe должен содержать `allow-scripts`.

### `alert()` / `confirm()` / `prompt()` не срабатывают (скрипт «как будто не работает»)
- **Причина:** модальные диалоги в песочнице блокируются без флага `allow-modals`.
  Скрипт выполняется, но вызов молча игнорируется (в консоли браузера — предупреждение
  «Ignored call to 'alert()'. ...the 'allow-modals' keyword is not set.»). В исходном
  ТЗ этот флаг забыли.
- **Решение:** добавить `allow-modals` в `sandbox`. Итоговый набор:
  `sandbox="allow-same-origin allow-scripts allow-popups allow-forms allow-modals"`.

### `</script>` в пользовательском JS рвёт документ iframe
- **Причина:** строка `</script>` внутри инлайнового `<script>` закрывает тег раньше времени.
- **Решение:** при сборке документа экранировать `</script>` → `<\/script>` в коде JS
  (и при желании в HTML), либо помнить про это ограничение. (Низкий приоритет для урока про цвет, но фиксируем.)

## Редактор: подсветка (оверлей) и Tab

### Текст ввода и подсветка «разъезжаются»
- **Причина:** у слоёв `.editor` (textarea) и `.highlight` (pre) разные метрики —
  шрифт, размер, `line-height`, `padding`, `tab-size`, рамка или `box-sizing`.
- **Решение:** задать ОБОИМ слоям одинаковые `font-family/size`, `line-height`,
  `padding`, `border:0`, `tab-size` (+ `-moz-tab-size`), `white-space: pre`,
  `letter-spacing: 0`, `box-sizing: border-box`. Оба `position:absolute; inset:0`.

### При прокрутке кода подсветка стоит на месте
- **Причина:** прокручивается только верхний `textarea`; нижний `pre` сам не двигается.
- **Решение:** на событие `scroll` у textarea синхронизировать
  `pre.scrollTop/scrollLeft = textarea.scrollTop/scrollLeft`. У `pre` `overflow:hidden`,
  `pointer-events:none` (чтобы клики шли к textarea).

### Подсветка ломает разметку / показывает теги как текст
- **Причина:** токены вставляются в `<code>` через `innerHTML`; неэкранированные
  `< > &` из кода ребёнка трактуются как теги.
- **Решение:** в токенайзере КАЖДЫЙ фрагмент (и совпавший токен, и «обычный» символ)
  прогонять через `escapeHtml` перед вставкой в HTML.

### Регэкспы токенайзера «съедают» текст не с той позиции
- **Причина:** обычный (нелипкий) регэксп ищет совпадение где угодно дальше по строке.
- **Решение:** использовать **липкий** флаг `/y` и перед каждым `exec` ставить
  `re.lastIndex = i`; тогда совпадение только строго с позиции `i`. Не забыть
  `&& m[0].length > 0`, иначе пустое совпадение зациклит проход.

### Tab уводит фокус вместо вставки табуляции
- **Причина:** по умолчанию Tab переключает фокус между элементами.
- **Решение:** на `keydown` при `e.key === 'Tab'` вызвать `e.preventDefault()` и
  вставить `\t`. Лучше через `document.execCommand('insertText', false, '\t')` —
  он сохраняет нативный Undo и сам шлёт `input`. Фолбаком — ручная правка `value`
  + `dispatchEvent(new Event('input'))`, иначе подсветка/автосохранение не обновятся.

## Emmet (свой мини-движок)

### Tab по аббревиатуре «съел» соседнюю готовую разметку
- **Причина:** при извлечении аббревиатуры слева от каретки `>` — это оператор Emmet,
  и наивный скан утаскивает кусок реального тега (`<br>` → «br>» → `<<br>>`).
- **Решение:** в `extractAbbreviation` останавливаться на пробеле/`\n`, а при встрече
  `<` считать, что задели разметку, и вернуть ПУСТУЮ аббревиатуру (→ обычный таб).
  Скобки `[] {} ()` учитывать счётчиком глубины (внутри них пробелы допустимы).

### Раскрытие «съедает» обычные слова или невалидный ввод
- **Причина:** Tab по любому тексту пытается раскрыть всё подряд.
- **Решение:** `isExpandable()` — раскрывать только если начинается с буквы/`.`/`#`/`(`,
  не содержит `<`, и либо есть структурные символы `>+^*.#[]{}()`, либо это слово —
  ИЗВЕСТНЫЙ HTML-тег (`KNOWN_TAGS`). Иначе обычный таб.

### После раскрытия слетает история отмены / не обновляется подсветка
- **Причина:** прямая запись `textarea.value` рвёт нативный Undo и не шлёт `input`.
- **Решение:** вставлять результат через `document.execCommand('insertText', …)` —
  он сохраняет Undo и сам диспатчит `input` (→ подсветка + автосохранение). Фолбэк —
  ручная правка + `dispatchEvent('input')`. Курсор ставить по метке из `expand()`,
  не забыв прибавить добавленные отступы строк (`newlines * pad.length`).

### Зацикливание токенайзера на пустом совпадении
- **Причина:** элемент-цикл не сдвинул `pos`.
- **Решение:** если внутренний цикл не продвинулся (`pos===start`) — `pos++` принудительно.
  Повторители `*N` ограничивать (`MAX_REPEAT`), иначе `div*999999` повесит вкладку.

### Превью-подсказка стоит не у курсора / «прыгает» при скролле
- **Причина:** в `<textarea>` нет API пиксельных координат курсора; наивно взять
  позицию строки нельзя. А ещё окно превью при прокрутке остаётся на месте.
- **Решение:** координаты курсора считать «зеркалом» — невидимый `div` с теми же
  метриками (`MIRROR_PROPS`: шрифт, padding, border, `tab-size`, `white-space:pre`),
  текст до курсора + span-маркер, читать `offsetLeft/offsetTop`. Экранная позиция =
  `textarea.getBoundingClientRect()` + offset − `scrollTop/Left`. На `scroll`
  пересчитывать позицию (или прятать). Клампить в окно; не влезает вниз → над строкой.

### Превью перехватывает клики/ввод
- **Причина:** окно поверх редактора ловит мышь и мешает печатать/выделять.
- **Решение:** `pointer-events: none` у `#emmet-preview`. Код внутри красить через
  `highlight()` (как в редакторе), вставлять во `<code>` (там уже есть `escapeHtml`).

## Встроенная консоль (мост iframe → платформа)

### `console.log` из кода ребёнка не виден на платформе
- **Причина:** код выполняется в изолированном iframe; его `console` пишет только в
  DevTools родителя нет доступа к нему напрямую.
- **Решение:** в документ iframe внедрить `CONSOLE_HOOK` ДО пользовательского скрипта —
  он подменяет `console.*` и шлёт `parent.postMessage({__console:true,…})`. Платформа
  слушает `message`. Требует `allow-same-origin` + `allow-scripts` в sandbox.

### Чужие/посторонние `message` попадают в консоль
- **Причина:** на `window` могут прилетать сообщения от расширений/других фреймов.
- **Решение:** в обработчике проверять `e.source === preview.contentWindow` и
  `e.data.__console === true`. Текст выводить через `textContent` (не `innerHTML`) —
  иначе вывод `console.log('<img onerror=…>')` исполнится в самой платформе (XSS).

### Консоль бесконечно растёт / показывает старый вывод
- **Причина:** строки копятся и не чистятся между запусками.
- **Решение:** лимит строк (срезать `firstElementChild`), автоскролл вниз,
  очистка при каждом «Запустить» (`updateIframe → clearConsole`).

## localStorage

### Исключение при доступе к localStorage (приватный режим / отключённый storage)
- **Причина:** в некоторых режимах обращение к `localStorage` кидает `SecurityError`.
- **Решение:** оборачивать чтение/запись в `try/catch`; при ошибке вести себя как «данных нет».

### После «Сброса» старый код возвращается после перезагрузки
- **Причина:** сбросили textarea, но не удалили ключ из localStorage.
- **Решение:** в `resetToTemplate()` обязательно `localStorage.removeItem('savedLessonCode')`
  (или перезапись initial-значениями) перед/после обновления редакторов.

## Скачивание проекта (ZIP вручную, без библиотек)

### Архив не открывается / «повреждён»
- **Причина:** неверный CRC32, рассинхрон размеров, неправильное смещение локального заголовка в центральном каталоге, или забытый EOCD.
- **Решение:** строго соблюдать формат. Сигнатуры: локальный `0x04034b50`, центральный `0x02014b50`, EOCD `0x06054b50`. Метод `0` (store), сжатый размер = исходному. `localOffset` каждого файла = его смещение **в потоке локальных заголовков**, считается ДО добавления центрального каталога. EOCD: `centralOffset` = суммарная длина всех локальных частей. Проверка: `zipfile.ZipFile(...).testzip()` должен вернуть `None`.

### Кракозябры в именах/содержимом с кириллицей
- **Причина:** ZIP по умолчанию трактует имена как CP437.
- **Решение:** ставить флаг бит 11 (`0x0800`) в general purpose flags локального и центрального заголовков — объявляет имена в UTF-8. Имена и данные кодировать через `TextEncoder` (UTF-8). Размеры брать в **байтах**, а не в `length` строки (символ ≠ байт).

### CRC32 без таблицы даёт неверный результат
- **Причина:** забыт начальный/финальный XOR с `~0`, или потеря старших битов (signed-сдвиги).
- **Решение:** `crc=~0; crc=(crc>>>1)^(0xEDB88320 & -(crc&1)); ... return (~crc)>>>0`. Беззнаковые сдвиги (`>>>`) обязательны.

## Скачивание файлов

### Память течёт / ссылка не скачивается
- **Причина:** не освобождён objectURL или ссылка не добавлена в DOM в некоторых браузерах.
- **Решение:** `URL.createObjectURL(blob)` → создать `<a>` с `download`, добавить в DOM,
  `click()`, удалить из DOM, затем `URL.revokeObjectURL(url)`.

## Прочее

### `defer` vs порядок инициализации
- **Причина:** если `main.js` без `defer` и подключён в `<head>`, DOM ещё не готов.
- **Решение:** подключать с `defer` или перед `</body>`; всю инициализацию — в обработчике `DOMContentLoaded`.
